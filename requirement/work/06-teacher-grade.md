# 06 · Teacher — ตรวจงาน (Grading)

> อ่าน `00-foundation.md` ก่อน · ภาพ: `../screens/teacher-grade.png` · บทบาท: **Teacher**

## Purpose
ดูโค้ดที่นักศึกษาส่ง + ผลตรวจ Test Case อัตโนมัติ + ให้คะแนนพิเศษ (bonus) + เขียน comment

## Layout
- **PageHead:** "ตรวจงาน" + คำบรรยาย "{ชื่อโจทย์} · {ชื่อวิชา}" + (ขวา) select **เลือกโจทย์**
- **กริด 3 คอลัมน์:** คิว `248px` / กลาง `1fr` / พาเนลให้คะแนน `320px`
  - <1280px → ซ่อนคอลัมน์คิว
  - <900px → ซ้อนแนวตั้ง

### ซ้าย — คิวรายการส่ง (`grade-queue`, sticky)
- หัว "รายการส่งงาน · {จำนวน}"
- รายการนักศึกษาที่ส่ง: Avatar(orange) + ชื่อ + รหัส + Badge สถานะ (ตรวจแล้ว=เขียว / รอตรวจ=เหลือง)
- รายการที่เลือกอยู่ = พื้น `--blue-50` + แถบซ้าย `--blue`

### กลาง — โค้ด + ผลตรวจ (`grade-center`)
**Code viewer (ธีมเข้ม):**
- แถบบน: จุด traffic-light 3 จุด + ชื่อไฟล์ `solution.py` + ภาษา (ขวา)
- เนื้อ: เลขบรรทัด (gutter) + โค้ด **syntax highlight Python** (keyword สีฟ้า `#7aa2ff`, comment สีเทา `#6b7794`)

**การ์ด "ผลการตรวจอัตโนมัติ":**
- หัว + Badge "ผ่าน X/Y เคส" (เขียวถ้าครบ, เหลืองถ้าไม่ครบ)
- รายการผลแต่ละเคส:
  - ผ่าน = กรอบ/พื้นเขียว ไอคอน ✓ วงกลมเขียว
  - ไม่ผ่าน = กรอบ/พื้นแดง ไอคอน ✗ วงกลมแดง + รายละเอียด (Input / คาดหวัง / ได้)
  - แต่ละแถวมีเวลา (ms) + คะแนน "ได้/เต็ม"
  - เคสที่ซ่อนมีป้าย "ซ่อน"

### ขวา — พาเนลให้คะแนน (`grade-panel`, sticky)
1. หัว: Avatar + ชื่อ + รหัส + Badge สถานะ
2. meta: เวลาส่ง (icon clock) + ภาษา (icon code) — คั่นเส้นบน-ล่าง
3. **คะแนนอัตโนมัติ (Test Case):** ตัวเลขใหญ่สี blue "auto/เต็ม" + progress bar
4. **คะแนนพิเศษ (Bonus)** — Field + **stepper** (ปุ่ม − / input number / ปุ่ม +); hint "เช่น คุณภาพโค้ด / ประสิทธิภาพ"
5. **คะแนนรวม** = auto + bonus (กล่องพื้น `--blue-50`, ตัวเลขใหญ่)
6. **ความคิดเห็นถึงนักศึกษา** (textarea)
7. ปุ่มเต็มกว้าง **"บันทึกผลการตรวจ"** (หรือ "บันทึกการแก้ไข" ถ้าตรวจแล้ว, icon check)

## Behavior
- เลือกนักศึกษาในคิว → เปลี่ยนโค้ด+ผล+พาเนลด้านขวา
- ปรับ stepper bonus → "คะแนนรวม" อัปเดตทันที
- กดบันทึก → เปลี่ยนสถานะ submission เป็น `graded`, เก็บ bonus + comment
- เลือกโจทย์อื่นจาก select บน → โหลดรายการส่งของโจทย์นั้น

## Logic / Data
- **Submission** (ดู 00): `{ id, sid, name, at, status, auto, bonus, pass, code, lang }`; คะแนนสุดท้าย = `auto + bonus`
- ผลต่อเคสในของจริงมาจาก backend (รัน sandbox เทียบ stdout กับ expected output, trim ท้ายบรรทัด). prototype จำลองด้วยจำนวนเคสที่ผ่าน (`pass`)
- รูปแบบการตรวจ: **auto จาก Test Case + bonus จากอาจารย์** (ตามที่ผู้ใช้เลือก)
