# 07 · Teacher — สมุดคะแนน (Scorebook รวม)

> อ่าน `00-foundation.md` ก่อน · ภาพ: `../screens/teacher-scores.png` · บทบาท: **Teacher**

## Purpose
ตารางคะแนนรวมของนักศึกษา **ทุกคน × ทุกโจทย์** ในรายวิชา

## Layout
- **PageHead:** "สมุดคะแนน" + คำบรรยาย "{รหัสวิชา} · {ชื่อวิชา}" + ปุ่ม **"ส่งออก Excel"** (outline sm, icon download)
- **การ์ด** บรรจุ legend + ตาราง (เลื่อนแนวนอนได้) + pagination

### Legend (เหนือตาราง)
จุดสี: เขียว=ส่งแล้ว · เหลือง=ส่งช้า · แดง=ค้างส่ง

### Table (เลื่อนแนวนอน — `table-scroll`)
| คอลัมน์ | รายละเอียด |
|---|---|
| # | อันดับ |
| รหัสนักศึกษา | mono |
| ชื่อ - นามสกุล | หนา |
| สถานะ | จุดสีเดียว (เขียว/เหลือง/แดง) |
| **(ต่อโจทย์)** | หนึ่งคอลัมน์ต่อโจทย์ — หัว = ชื่อโจทย์ + คะแนนเต็ม (เล็ก) |
| รวม | คอลัมน์ท้าย เน้น (หัวพื้น `--blue-100`) |

- เซลล์คะแนนต่อโจทย์ = **pill คะแนน** สีตามสัดส่วน:
  - `hi` (≥80% ของเต็ม) = เขียว · `mid` (≥50%) = เหลือง · `lo` (<50%) = แดง
  - "**–**" (muted) ถ้ายังไม่ส่ง (`null`)
- เซลล์ "รวม" = ผลรวมคะแนนของแถวนั้น (เน้นสี blue-d), พื้นคอลัมน์จาง
- หัวตารางพื้น `--blue-50`

### Pagination
ใต้ตาราง (เหมือนหน้าอื่น)

## Behavior
- "ส่งออก Excel" → ดาวน์โหลดไฟล์คะแนน (ของจริง: gen .xlsx ฝั่ง client/backend)
- (ออปชัน) คลิกหัวคอลัมน์เพื่อ sort, คลิกแถวเพื่อดูรายละเอียดนักศึกษา — ปรับเพิ่มได้

## Data
**Scorebook row** (ดู 00): `{ rank, sid, name, scores:(number|null)[], status }` — `scores` เรียงตามลำดับโจทย์ (index ตรงกับ PROBLEMS); หัวคอลัมน์ต่อโจทย์ดึงจากรายการโจทย์ (ชื่อ + `points`)
