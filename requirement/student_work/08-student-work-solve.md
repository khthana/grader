## A) งานที่ได้รับมอบหมาย — `student-work.png`

นักศึกษา **เลือกสัปดาห์ก่อน** แล้วดูงานของสัปดาห์นั้น เป็น **ตาราง** (เลย์เอาต์เดียวกับหน้าโจทย์ของอาจารย์)

### Layout
- **PageHead:** "งานที่ได้รับมอบหมาย" + คำบรรยาย "{รหัสวิชา} · {ชื่อวิชา}"
- **WeekBar** — เหมือนไฟล์ 04 (ป้าย "สัปดาห์" + ปุ่มสัปดาห์ 2 บรรทัด, active พื้น blue)
- **การ์ด:** section-head ("สัปดาห์ที่ N · {หัวข้อ}" + "N งาน") + ตาราง

### Table
| คอลัมน์ | รายละเอียด |
|---|---|
| ลำดับ | ในสัปดาห์ |
| ชื่อโจทย์ | หนา |
| กำหนดส่ง | muted nowrap |
| คะแนนเต็ม | mono |
| สถานะ | Badge — ตรวจแล้ว(+คะแนน)=เขียว / ส่งแล้ว·รอตรวจ=น้ำเงิน / ยังไม่ส่ง=เหลือง |
| (action) | ปุ่ม **"ทำโจทย์"** (primary, ถ้ายังไม่ส่ง) หรือ **"ดูงาน"** (outline) |

- **⚠️ ไม่มี Level**, เป็นตาราง (ไม่ใช่การ์ด)
- สัปดาห์ว่าง → แถว empty "ยังไม่มีงานในสัปดาห์นี้"

### Behavior
เลือกสัปดาห์ → เปลี่ยนรายการ. คลิกปุ่ม → เปิดหน้า **Solve** (sub-view เต็มหน้า)

## B) ทำโจทย์ (Solve) — `student-solve.png`

### Layout (ค่าเริ่มต้น "split")
- บนสุด: ปุ่ม **"กลับไปงานที่ได้รับ"** (+ Badge "ส่งแล้ว" ถ้าเคยส่ง)
- **กริด 2 คอลัมน์:** โจทย์ (ซ้าย) / เอดิเตอร์+ผล (ขวา). ที่ <1000px ยุบเป็นคอลัมน์เดียว
- (มี tweak เดโม: `stacked` = บน-ล่าง, `tabs` = สลับแท็บ — ของจริงเลือก split เป็นหลักพอ)

### โจทย์ (ซ้าย, การ์ดขาว, เลื่อนได้)
- หัว: "โจทย์ N" + คะแนน (ขวา) — **ไม่มี badge ระดับ**
- ชื่อโจทย์ (ใหญ่หนา) → คำอธิบาย
- บล็อก "รูปแบบ Input" / "รูปแบบ Output" (pre พื้น `--bg`)
- "ตัวอย่าง": คู่ Input/Output (pre พื้นเข้ม) — **เฉพาะ test ที่ไม่ซ่อน**

### เอดิเตอร์ (ขวา, ธีมเข้ม)
- แถบบน: ภาษา "Python 3.11" + ปุ่ม **"รันทดสอบ"** (outline) + **"ส่งคำตอบ"** (primary)
- พื้นที่โค้ด: textarea + เลขบรรทัด; **Tab = 4 ช่องว่าง**; ฟอนต์ mono ตัวอักษรอ่อนบนพื้นเข้ม
- พื้นที่ผลลัพธ์ (ล่าง):
  - idle: ข้อความชวนกด รัน/ส่ง
  - running: สปินเนอร์ "กำลังรันโค้ดและตรวจ Test Case…"
  - results: **verdict** (เขียว=ผ่าน / แดง=ไม่ผ่าน) + รายการผลต่อเคส (ไอคอน ✓/✗, ชื่อ, เวลา ms, คะแนน)

### Behavior
- **"รันทดสอบ":** รันกับ **เฉพาะ test ที่ไม่ซ่อน** → แสดงผลต่อเคส (ไม่บันทึกการส่ง)
- **"ส่งคำตอบ":** รัน **ทุก test (รวมที่ซ่อน)** → แสดง verdict + คะแนนรวม, เปลี่ยนสถานะเป็น "ส่งแล้ว"
- ของจริง: ส่งโค้ดไป backend รันใน sandbox; prototype จำลองผล (สปินเนอร์ ~0.9s แล้วตรวจสตริงในโค้ด)
